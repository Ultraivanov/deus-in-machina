# Release Checklist

## Buildrail — Pre‑Release Steps

1. Review security checklist: `docs/security-checklist.md`.
2. Review privacy notes: `docs/privacy-notes.md`.
3. Review dependency audit: `docs/dependency-audit.md`.
4. Confirm onboarding + FAQ: `docs/onboarding-checklist.md`, `docs/faq.md`.
5. Verify packaging guide: `docs/packaging-guide.md`.
6. Verify hosted deployment outline: `docs/hosted-deployment.md`.
7. Run smoke flow (initialize → next_step → start_session → submit → validate → complete).
8. Tag release and update README if needed.
