# Block E-01: Organizations + User Management

> Enterprise foundation for multi-tenant DSR adoption.

---

## Goal

Enable 15+ users to collaborate in DSR without conflicts through organization-based multi-tenancy with role-based access control.

---

## Success Criteria (DoD)

- [ ] User can create organization
- [ ] Owner can invite users by email
- [ ] Users have roles: Owner, Admin, Member
- [ ] Resources isolated by organization
- [ ] API endpoints protected by org membership
- [ ] Tests: 90%+ coverage for auth layer

---

## Tasks

### T1: Database Schema
**Scope:** PostgreSQL schema for multi-tenancy

- [ ] Create `organizations` table
  - id (UUID), name, slug, created_at, updated_at
  - billing_plan (free/pro/enterprise)
  - quotas: max_users, max_workspaces, max_tokens
- [ ] Create `users` table
  - id (UUID), email (unique), password_hash
  - email_verified_at, created_at, updated_at
- [ ] Create `organization_members` junction
  - org_id, user_id, role (owner/admin/member)
  - invited_at, joined_at
- [ ] Add foreign keys and indexes
- [ ] Migration script with rollback

**Done When:** `npx migrate up` creates tables, `npx migrate down` removes them

---

### T2: Auth API Endpoints
**Scope:** REST API for authentication

- [ ] POST `/api/v1/auth/register`
  - email, password, org_name
  - Creates user + org (as owner)
- [ ] POST `/api/v1/auth/login`
  - email, password
  - Returns JWT access + refresh tokens
- [ ] POST `/api/v1/auth/refresh`
  - refresh_token → new access token
- [ ] POST `/api/v1/auth/logout`
  - Invalidate refresh token
- [ ] POST `/api/v1/auth/forgot-password`
  - Send reset email (mock SMTP for now)
- [ ] POST `/api/v1/auth/reset-password`
  - token + new_password

**Done When:** All endpoints return correct status codes, JWT tokens valid

---

### T3: Organization API
**Scope:** Org management endpoints

- [ ] GET `/api/v1/orgs` - List user's organizations
- [ ] GET `/api/v1/orgs/:id` - Get org details
- [ ] PATCH `/api/v1/orgs/:id` - Update org (owner/admin only)
  - name, billing_plan
- [ ] DELETE `/api/v1/orgs/:id` - Delete org (owner only)

- [ ] POST `/api/v1/orgs/:id/invite` - Invite user
  - email, role (default: member)
  - Sends invitation email
- [ ] POST `/api/v1/orgs/:id/invites/:token/accept` - Accept invite
- [ ] DELETE `/api/v1/orgs/:id/members/:userId` - Remove member
- [ ] PATCH `/api/v1/orgs/:id/members/:userId` - Change role

**Done When:** All endpoints protected by role middleware, tests pass

---

### T4: Middleware & Protection
**Scope:** Auth layer for all API routes

- [ ] `authMiddleware` - Validate JWT, attach user to req
- [ ] `requireAuth` - Reject 401 if no valid token
- [ ] `requireRole(roles[])` - Check org membership + role
- [ ] `requireOrgAccess` - Verify user in org
- [ ] Error responses consistent format
  ```json
  { "error": "Unauthorized", "code": "AUTH_INVALID_TOKEN" }
  ```

**Done When:** All protected routes return 401/403 correctly

---

### T5: Integration with Existing Resources
**Scope:** Add org_id to current DSR entities

- [ ] Add `org_id` to tokens table
- [ ] Add `org_id` to patterns table  
- [ ] Add `org_id` to validations table
- [ ] Update all queries to filter by org_id
- [ ] Ensure isolation: User A cannot see User B's tokens

**Done When:** Multi-org deployment works, no data leakage

## API Contract (OpenAPI)

```yaml
# Document in docs/api-v1.yaml
paths:
  /auth/register:
    post:
      summary: Register new user and create org
      requestBody:
        content:
          application/json:
            schema:
              type: object
              required: [email, password, org_name]
              properties:
                email: { type: string, format: email }
                password: { type: string, minLength: 8 }
                org_name: { type: string, minLength: 2 }
      responses:
        201: { description: User and org created }
        400: { description: Validation error }
        409: { description: Email already exists }
```

## Security Considerations

- Passwords: bcrypt with salt rounds 10
- JWT: HS256, 15min access, 7d refresh
- Rate limiting: 100 req/min per IP
- CORS: Whitelist frontend domains
- Helmet.js for security headers
