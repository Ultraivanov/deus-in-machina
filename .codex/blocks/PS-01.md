## Block Goal

Make the released package and runtime smoke paths observable and reproducible so post-release breakages are caught by explicit verification instead of slipping past unit tests.

## Definition of Done

A documented smoke suite runs against the shipped package surface and core runtime scripts, fails on the currently confirmed regressions, and produces a stable checklist of post-release failures to drive repair blocks.

### PS-01-T1 — Define release smoke matrix

**Files to modify:**
- `.codex/blocks/PS-01.md` — mark task in-progress and log plan

**Files to create:**
- `docs/release-smoke-matrix.json` — machine-readable smoke check manifest
- `docs/release-smoke-matrix.md` — human-readable smoke check documentation

**Files NOT touched:**
- `src/`
- `scripts/`
- `package.json`
- `README.md`
- fixtures and expected outputs

**Approach:**
Define a stable smoke-check contract first, before writing any runner code. Encode
the release checks as a small JSON matrix with ids, commands, expected exit behavior,
and purpose, then add a short Markdown companion that explains the checks and records
the currently confirmed regressions this matrix is meant to expose.

**Risks:**
If the matrix format is too ad hoc, `PS-01-T2` will have to reshape it. Keep it
minimal and command-oriented so the runner can consume it directly.
