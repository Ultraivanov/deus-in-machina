## Block Goal

Enhance the ruleset system with configurable rule packs, severity levels, and flexible validation policies to support different project requirements and compliance levels.

## Definition of Done

- Rule packs organized by category (tokens, layout, accessibility, etc.)
- Severity levels configurable per rule (error, warning, info, ignore)
- Rule enable/disable configuration
- Preset rule packs (strict, relaxed, minimal)
- Custom rule pack creation support
- Validation engine uses severity from ruleset
- Documentation for rule pack configuration

> One entry per task. Written by agent before coding. Approved by user before execution.

### B-03-T1-T5 Change Plan (Approved & Executed)

Implemented configurable rule packs and severity system:

1. **Created `src/ruleset/categories.js`** — Rule categorization:
   - 6 categories: tokens, spacing, patterns, layout, accessibility, performance
   - CORE_RULES registry with metadata (id, name, description, category, severity)
   - Helper functions: getRulesByCategory, getEnabledRules, getRule

2. **Updated `src/ruleset/contract.js`** — Severity levels:
   - Added 'ignore' to Severity type
   - Added documentation for each severity level

3. **Created `src/ruleset/presets.js`** — Preset rule packs:
   - STRICT_RULESET — Maximum enforcement (all errors)
   - RELAXED_RULESET — Balanced for development
   - MINIMAL_RULESET — Only critical issues
   - A11Y_RULESET — Accessibility-focused
   - PERF_RULESET — Performance-focused
   - extendPreset() for custom rulesets

4. **Updated `src/ruleset/loader.js`** — Custom pack support:
   - Preset loading integrated into loadRuleset()
   - isValidRuleset() for validation

5. **Updated `src/validator.js`** — Engine integration:
   - isIgnored() check skips validation for ignored severity
   - Category metadata added to all issues
   - byCategory grouping in results
   - Severity info in meta output

6. **Updated `src/ruleset/index.js`** — Exports all new modules
