- `bin/`
- `package.json`
- `scripts/`
- unrelated untracked enterprise duplicates

**Approach:**
Document the export surface exactly as it exists today, not as it should work. Derive
the contract from `package.json`, map every declared export to the actual file present
in `src/`, and explicitly mark broken declarations such as `.` -> `src/index.js` when
only `src/index.ts` exists.

**Verification:**
- every `package.json` export is represented in the contract
- the contract distinguishes `declared path`, `actual file present`, and `status`
- root package entrypoint and CLI entrypoint are both covered
