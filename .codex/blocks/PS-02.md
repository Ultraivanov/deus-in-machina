### PS-02-T1 — Define runtime export contract

**Files to modify:**
- `.codex/blocks/PS-02.md` — mark task in-progress and log plan

**Files to create:**
- `docs/package-export-contract.md` — canonical package surface document listing root entrypoint, CLI entrypoint, declared subpath exports, current target files, and confirmed mismatches against `package.json`

**Files NOT touched:**
- `src/`
- `bin/`
- `package.json`
- `scripts/`
- unrelated untracked enterprise duplicates

**Approach:**
Document the export surface exactly as it exists today, not as it should work. Derive
the contract from `package.json`, map every declared export to the actual file present
in `src/`, and explicitly mark broken declarations such as `.` -> `src/index.js` when
only `src/index.ts` exists.

**Verification:**
- every `package.json` export is represented in the contract
- the contract distinguishes `declared path`, `actual file present`, and `status`
- root package entrypoint and CLI entrypoint are both covered

### PS-02-T2 — Repair root entrypoint and package exports

**Files to modify:**
- `.codex/blocks/PS-02.md` — mark task in-progress and record the approved Change Plan

**Files to create:**
- `src/index.js` — real root package entrypoint that exports the current JS runtime surface
- `tests/package-exports.test.js` — regression test that imports the root entrypoint and verifies the expected public symbols exist

**Files NOT touched:**
- `package.json`
- `bin/dsr.js`
- `scripts/build.js`
- async runner scripts
- unrelated untracked enterprise duplicates

**Approach:**
The current break is structural: `package.json` already points to `src/index.js`, but
that file does not exist. The least risky repair is to create `src/index.js` as the
canonical JS entrypoint and re-export the stable runtime modules that already exist in
`.js` form, such as normalizer, patterns, validator, fix loop, ruleset, and Figma
adapter helpers.

**Verification:**
- `node -e "import('./src/index.js')"` succeeds
- the root entrypoint exposes the intended named exports
- `npm test` includes the new regression test and stays green
