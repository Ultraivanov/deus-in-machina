### I-05-T2 — Rule checks (token + spacing + pattern)

**Files to modify:**
- `src/validator.js` — add deterministic rule checks

**Files to create:**
- none

**Files NOT touched:**
- CLI, workflow engine, adapters, normalizer, patterns, fixes, monetization, stripe, storage

**Approach:**
Implement three simple checks on `code` (string-level):
1) raw hex colors → `dsr.token.no-raw-values` (ERROR)
2) margin/padding not divisible by 8 → `dsr.spacing.grid-8pt` (ERROR)
3) hero without cta → `dsr.pattern.hero-missing-cta` (WARNING)
Compute `summary` counts and set `valid` when errors = 0.

**Risks:**
Text heuristics are simplistic and may evolve later for structured parsing.
